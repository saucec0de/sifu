Consider the following four implementations of getting a greeting (in C): 

  1. char* get_morning_greeting(bool english) {
    if (english)
        return "Good morning!";
    else
        return "Guten Morgen!";
}


  2. char* get_morning_greeting(bool english) {
    char english_greeting[] = "Good morning!";
    char german_greeting[] = "Guten Morgen!";
    
    if (english)
        return english_greeting;
    else
        return german_greeting;
}


  3. char* morning_greeting;
void get_morning_greeting(bool english) {
    char english_greeting[] = "Good morning!";
    char german_greeting[] = "Guten Morgen!";
    
    if (english)
        morning_greeting = english_greeting;
    else
        morning_greeting = german_greeting;
}


  4. char* get_morning_greeting(bool english) {
    static char english_greeting[] = "Good morning!";
    static char german_greeting[] = "Guten Morgen!";
    
    if (english)
        return english_greeting;
    else
        return german_greeting;
}

Suppose we use one of the above implementations to get a greeting and print it to the user. Using which one of the above implementations will not cause undefined behaviour?

Note: in order to use implementation 3, we call the function and then print the global variable. For the other implementations, we directly print the return value.
Assume stdbool.h has been imported.
