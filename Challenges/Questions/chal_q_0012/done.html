The problem with the other implementation was that the returned string could be modified. Of course, it is true that in the first implementation modifying the returned string would lead to undefined behavior which is bad. However, in practice, this will cause the program to crash (since string literals are generally allocated in the read-only data section) which will provide the user calling the function a useful diagnostic that there is something wrong with their code.


One more thing: it is a good idea to use a const-qualifier for the return type to indicate that the string should not be modified. (But beware of "const-poisoning" when working with existing code.)

const char* greet_in_morning(bool english) {
    if (english)
        return "Good morning!";
    else
        return "Guten Morgen!";
}

See STR05-C. Use pointers to const when referring to string literals.
